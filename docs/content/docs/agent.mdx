The Obidot AI Agent is an autonomous financial strategist that manages the vault system. Built with TypeScript, [LangChain](https://js.langchain.com/), and [Viem](https://viem.sh/), it runs an observation-decision-execution loop that monitors yield opportunities across Polkadot parachains, then signs and submits strategy intents to the vault's on-chain policy engine. It also exposes a Fastify HTTP/WebSocket API consumed by the Next.js dashboard, and optionally integrates with Telegram.

## Decision Types

The agent produces exactly one of six decision types per cycle:

### REALLOCATE — XCM Parachain Strategy

Routes capital to a Polkadot parachain via the existing XCM strategy pipeline:

```json
{
  "action": "REALLOCATE",
  "targetParachain": 2034,
  "targetProtocol": "0x1234...",
  "amount": "1000000000000000000",
  "maxSlippageBps": 50,
  "reasoning": "Hydration Omnipool APY 12.3% vs current 8.1%, net benefit after XCM cost is 3.95%"
}
```

### BIFROST_STRATEGY — Bifrost DeFi Operation

Executes a specific Bifrost DeFi operation (liquid staking, DEX swap, farming, etc.):

```json
{
  "action": "BIFROST_STRATEGY",
  "strategyType": 0,
  "amount": "500000000000000000",
  "maxSlippageBps": 30,
  "currencyIn": 0,
  "currencyOut": 1,
  "reasoning": "Mint vDOT via SLP: 9.2% staking yield, stable and low-risk"
}
```

| Field          | Type     | Description                                             |
| -------------- | -------- | ------------------------------------------------------- |
| `strategyType` | `0–6`    | BifrostStrategyType enum value                          |
| `currencyIn`   | `0–4`    | Input currency ID (DOT=0, vDOT=1, KSM=2, vKSM=3, BNC=4) |
| `currencyOut`  | `0–4`    | Output currency ID (optional, for swaps)                |
| `poolId`       | `number` | Farming pool ID (optional, for farm operations)         |
| `minOutput`    | `string` | Minimum expected output in wei (optional)               |

### CROSS_CHAIN_REBALANCE — Hub ↔ Satellite

Proposes moving liquidity between the hub vault and satellite vaults (logged for manual execution):

```json
{
  "action": "CROSS_CHAIN_REBALANCE",
  "targetChain": "ethereum",
  "direction": "HUB_TO_SATELLITE",
  "amount": "2000000000000000000",
  "reasoning": "Ethereum satellite has high withdrawal demand, rebalance 2 ETH to improve liquidity"
}
```

### LOCAL_SWAP — On-Hub DEX Aggregator Swap

Routes a swap through the on-hub `SwapRouter` via `vault.executeLocalSwap()`:

```json
{
  "action": "LOCAL_SWAP",
  "tokenIn": "0x2402C804aD8a6217BF73D8483dA7564065c56083",
  "tokenOut": "0xAf233E9f2ED78022CAdEA58a84144ce6BcDFd63E",
  "amount": "500000000000000000",
  "minOut": "495000000",
  "reasoning": "Rebalance DOT→USDC: local swap via SwapRouter, 0.1% estimated slippage"
}
```

### UNIVERSAL_INTENT — Cross-Chain Intent (XCM or Hyperbridge)

Signs and submits a `UniversalIntent` via `vault.executeIntent()`. Supports both XCM (destType=0, parachains) and Hyperbridge (destType=1, EVM chains):

```json
{
  "action": "UNIVERSAL_INTENT",
  "destType": 0,
  "paraId": 2034,
  "chainId": 0,
  "inToken": "0x2402C804aD8a6217BF73D8483dA7564065c56083",
  "amount": "1000000000000000000",
  "minOut": "990000000",
  "reasoning": "XCM swap DOT→HDX on Hydration Omnipool via UniversalIntent, destType=Native"
}
```

### NO_ACTION — Hold Position

When no profitable opportunity exists or risk is too high:

```json
{
  "action": "NO_ACTION",
  "reasoning": "All APY deltas below 2% threshold after costs. Current positions are optimal."
}
```

## LangChain Tools

The agent registers 9 custom LangChain tools:

### Perception Tools

| Tool                       | Name                      | Purpose                                                                |
| -------------------------- | ------------------------- | ---------------------------------------------------------------------- |
| `FetchYieldsTool`          | `fetch_yields`            | Fetches APY rates from all tracked protocols (parachains + Bifrost)    |
| `FetchVaultStateTool`      | `fetch_vault_state`       | Reads on-chain vault state (idle balance, remote assets, caps, health) |
| `FetchBifrostYieldsTool`   | `fetch_bifrost_yields`    | Fetches yield data for all 7 Bifrost products                          |
| `FetchCrossChainStateTool` | `fetch_cross_chain_state` | Aggregates state from hub + all satellite EVM vaults                   |
| `SwapQuoteTool`            | `swap_quote`              | Calls `SwapQuoter.getBestQuote()` for a token pair and amount          |

### Execution Tools

| Tool                         | Name                       | Purpose                                                                      |
| ---------------------------- | -------------------------- | ---------------------------------------------------------------------------- |
| `ExecuteStrategyTool`        | `execute_strategy`         | Signs and submits a REALLOCATE strategy intent via EIP-712                   |
| `ExecuteBifrostStrategyTool` | `execute_bifrost_strategy` | Executes a Bifrost DeFi operation via BifrostAdapter                         |
| `ExecuteLocalSwapTool`       | `execute_local_swap`       | Calls `vault.executeLocalSwap()` for on-hub DEX aggregator swaps             |
| `ExecuteIntentTool`          | `execute_intent`           | Signs and submits a `UniversalIntent` via `vault.executeIntent()` (XCM/ISMP) |

### Tool Guardrails

All execution tools enforce hard limits before submitting transactions:

- **Maximum amount**: Capped at `MAX_STRATEGY_AMOUNT` (configurable, default 100 ETH equivalent)
- **Maximum slippage**: Capped at 100 bps (1%)
- **Action type validation**: Each tool rejects decisions with the wrong `action` type
- **Zod schema validation**: All decisions are parsed through the `aiDecisionSchema` discriminated union

## Services

### YieldService

Fetches yield data from DeFi protocols using a fast per-slug TVL approach:

- **TVL data**: Fetched from `api.llama.fi/tvl/{slug}` per protocol — returns a single number in ~700ms (replaces the bulk `/yields/pools` endpoint which returns ~12MB and always times out)
- **APY data**: Simulated via a sine-wave oscillator seeded per protocol, producing realistic bounded ranges
- **Bifrost yields**: 7 products returned with simulated APYs (Bifrost's API returns collator data, not vToken APYs)
- **Cache TTL**: 5 minutes (previously 2 minutes before the DeFiLlama fix)

Tracked parachain protocols and their APY ranges:

| Protocol           | Parachain       | APY Range |
| ------------------ | --------------- | --------- |
| Hydration Omnipool | HydraDX (2034)  | 8–15%     |
| Acala Liquid DOT   | Acala (2000)    | 6–12%     |
| Moonwell Lending   | Moonbeam (2004) | 4–9%      |
| Astar dApp Staking | Astar (2006)    | 5–10%     |

Bifrost products:

| Product                  | Category | APY Range |
| ------------------------ | -------- | --------- |
| Bifrost SLP vDOT         | SLP      | 7–11%     |
| Bifrost SLP vKSM         | SLP      | 5.5–8.5%  |
| Bifrost DEX DOT/vDOT     | DEX      | 8–18%     |
| Bifrost DEX BNC/DOT      | DEX      | 12–25%    |
| Bifrost Farm DOT/vDOT LP | Farming  | 15–35%    |
| Bifrost Farm BNC/DOT LP  | Farming  | 20–45%    |
| Bifrost SALP             | SALP     | 3–6%      |

### CrossChainService

Aggregates vault state across all configured chains:

```typescript
class CrossChainService {
  async fetchCrossChainState(
    hubState: VaultState,
  ): Promise<CrossChainVaultState>;
  getStaleSatellites(): SatelliteChainState[];
  async isRouterActive(): Promise<boolean>;
}
```

Reads performed in parallel for each configured satellite:

| Read                 | Source                       | Data                                                          |
| -------------------- | ---------------------------- | ------------------------------------------------------------- |
| Hub cross-chain data | `ObidotVault`                | `totalSatelliteAssets`, `globalTotalAssets`                   |
| Satellite state      | `ObidotVaultEVM` (per chain) | `totalAssets`, `emergencyMode`, `lastSyncTimestamp`, `paused` |
| Router status        | `CrossChainRouter`           | `paused`                                                      |

### SignerService

Manages EIP-712 signing and on-chain transaction execution:

| Method                              | Purpose                                                 |
| ----------------------------------- | ------------------------------------------------------- |
| `signStrategyIntent(intent)`        | EIP-712 typed data signing for vault strategy intents   |
| `executeOnChain(intent, signature)` | Submit signed intent to `ObidotVault.executeStrategy()` |
| `previewBifrostStrategy(...)`       | Read `BifrostAdapter.previewStrategy()`                 |
| `executeBifrostStrategy(...)`       | Write `BifrostAdapter.executeBifrostStrategy()`         |

Uses Viem's `createWalletClient` with the strategist's private key for signing and `createPublicClient` for reading.

### SwapRouterService

Reads swap quotes from the on-hub DEX aggregator and constructs optimal routes:

| Method                                  | Purpose                                                           |
| --------------------------------------- | ----------------------------------------------------------------- |
| `getBestQuote(tokenIn, tokenOut, amt)`  | Calls `SwapQuoter.getBestQuote()` — returns best adapter + amount |
| `getAllQuotes(tokenIn, tokenOut, amt)`  | Calls `SwapQuoter.getAllQuotes()` — all adapters ranked by output |
| `quoteMultiHop(path, amounts)`          | Calls `SwapQuoter.quoteMultiHop()` — multi-hop route quote        |
| `buildBestSwap(tokenIn, tokenOut, amt)` | Calls `SwapQuoter.buildBestSwap()` — returns `SwapParams` struct  |
| `supportsPair(adapter, a, b)`           | Calls `IPoolAdapter.supportsPair()` on a specific adapter         |

Configured via `SWAP_QUOTER_ADDRESS`, `SWAP_ROUTER_ADDRESS`, and optional per-adapter addresses in `.env`. Gracefully skips when addresses are not set.

### StrategyStore

In-memory singleton that records all strategy executions and agent decisions for the API:

```typescript
class StrategyStoreService {
  addStrategy(record: StrategyRecord): void;
  updateStrategy(id: string, update: Partial<StrategyRecord>): void;
  getStrategies(): StrategyRecord[];
  addDecision(decision: AgentDecisionRecord): void;
  getDecisions(limit?: number): AgentDecisionRecord[];
}
```

`StrategyRecord` fields: `id`, `action`, `target`, `amount`, `reasoning`, `status` (`pending` | `executed` | `failed` | `timeout`), `timestamp`, `outcomeTimestamp`, `txHash`.

`AgentDecisionRecord` fields: `cycle`, `action`, `reasoning`, `timestamp`, plus an optional `snapshot` (vault state at decision time: `totalAssets`, `idleBalance`, `topYieldApy`, `topYieldProtocol`).

### EventBus

A Node.js `EventEmitter` singleton used to relay real-time events to WebSocket clients:

```typescript
class EventBusService {
  emit(event: string, data: unknown): void;
  on(event: string, listener: (data: unknown) => void): void;
  off(event: string, listener: (data: unknown) => void): void;
}
```

Event types currently emitted: `vault:stateUpdate`, `strategy:executed`, `agent:decision`, `heartbeat`.

### OracleService

Multi-source price aggregation with TWAP (time-weighted average price):

```typescript
class OracleService {
  async getPrice(symbol: string): Promise<OracleQuote>;
  async getAllPrices(): Promise<Map<string, OracleQuote>>;
  async startPriceUpdates(intervalMs?: number): Promise<void>;
}
```

Configured via `oracle.config.ts` which defines feed symbols, decimal precision, and heartbeat intervals for each asset.

### PriceAggregatorService

Fetches prices from multiple off-chain sources and aggregates them:

| Source      | Endpoint              | Use                                |
| ----------- | --------------------- | ---------------------------------- |
| Pyth Hermes | `hermes.pyth.network` | Primary price feed for most assets |
| CoinGecko   | `api.coingecko.com`   | Fallback and cross-reference       |
| Binance     | `api.binance.com`     | Spot price cross-check             |
| Subsquid    | GraphQL endpoint      | On-chain event indexing            |

## Autonomous Loop

### Phase 1: Perception

```typescript
const yields = await yieldService.fetchYields();
const bifrostYields = await yieldService.fetchBifrostYields();
const vaultState = await fetchVaultStateTool.invoke();
const crossChain = await crossChainService.fetchCrossChainState(vaultState);
```

### Phase 2: Decision

```typescript
const decision = await llm.invoke([
  { role: "system", content: SYSTEM_PROMPT },
  {
    role: "user",
    content: buildUserMessage({
      yields,
      bifrostYields,
      vaultState,
      crossChain,
    }),
  },
]);
```

The user message includes APY rates, Bifrost product yields, vault balances, and cross-chain satellite states.

### Phase 3: Execution

| Action                  | Execution                                                                           |
| ----------------------- | ----------------------------------------------------------------------------------- |
| `REALLOCATE`            | `execute_strategy` tool → EIP-712 sign → `vault.executeStrategy()`                  |
| `BIFROST_STRATEGY`      | `execute_bifrost_strategy` tool → `adapter.executeBifrostStrategy()`                |
| `LOCAL_SWAP`            | `execute_local_swap` tool → `vault.executeLocalSwap()` via SwapRouter               |
| `UNIVERSAL_INTENT`      | `execute_intent` tool → EIP-712 sign → `vault.executeIntent()` (XCM or Hyperbridge) |
| `CROSS_CHAIN_REBALANCE` | Logged to strategy store — requires manual admin execution                          |
| `NO_ACTION`             | Logged to strategy store, no transaction                                            |

### Phase 4: Outcome Recording

After each cycle the decision (action, reasoning, snapshot) is written to `StrategyStoreService` and emitted on the `EventBus` so WebSocket clients receive it in real-time.

## API Server

The Fastify HTTP server starts on port `API_PORT` (default `3011`). All routes are prefixed `/api/`. CORS is open (suitable for local development). WebSocket upgrades are handled at `GET /ws`.

### Endpoints

| Method | Path                     | Description                                                         |
| ------ | ------------------------ | ------------------------------------------------------------------- |
| `GET`  | `/api/vault/state`       | On-chain vault state from `ObidotVault`                             |
| `GET`  | `/api/vault/performance` | Cumulative PnL, high-water mark, fees accrued                       |
| `GET`  | `/api/yields`            | All parachain protocol yields                                       |
| `GET`  | `/api/yields/bifrost`    | All Bifrost product yields                                          |
| `GET`  | `/api/strategies`        | Strategy execution history from StrategyStore                       |
| `GET`  | `/api/crosschain/state`  | Hub + satellite vault state aggregate                               |
| `GET`  | `/api/agent/log`         | Agent decision log (most recent first)                              |
| `POST` | `/api/agent/trigger`     | Trigger an LLM cycle — GPT-4o with all tools (up to 5 iterations)   |
| `GET`  | `/api/swap/quote`        | Best swap quote for a token pair via `SwapQuoter.getBestQuote()`    |
| `GET`  | `/api/swap/routes`       | All adapter quotes for a token pair via `SwapQuoter.getAllQuotes()` |
| `GET`  | `/ws`                    | WebSocket — streams `EventBus` events to connected clients          |

### Response Envelope

All REST responses follow a standard envelope:

```json
{ "success": true, "data": <T> }
{ "success": false, "error": "message" }
```

### WebSocket Events

The WebSocket streams `EventBus` events as JSON:

```typescript
type WsEvent =
  | { type: "vault:stateUpdate"; data: VaultState }
  | { type: "strategy:executed"; data: StrategyRecord }
  | { type: "agent:decision"; data: AgentDecisionRecord }
  | { type: "heartbeat"; data: { timestamp: string } };
```

The heartbeat fires every 30 seconds to keep connections alive.

## Telegram Bot

When `TELEGRAM_BOT_TOKEN` is set, a Telegram bot is started alongside the loop and API server. The bot:

- Responds to the `/start` command with a welcome message
- Passes any text message to GPT-4o with all 9 LangChain tools available
- Runs up to 5 tool-calling iterations per query (same runner as `/api/agent/trigger`)
- Uses `ADMIN_CHAT_ID` to restrict access to a specific Telegram user/chat (optional)

## System Prompt

The system prompt defines the agent's identity and constraints:

### Identity

> You are the **Obidot Autonomous CFO** — an AI financial strategist operating across Polkadot and EVM chains.

### Risk Rules (Hard Constraints)

1. Maximum slippage: never above 100 bps (1%)
2. APY delta threshold: only act if delta exceeds configured minimum
3. XCM cost awareness: factor ~0.1% per transfer
4. ISMP cost awareness: factor ~0.25% per Hyperbridge message
5. Capital preservation: NO_ACTION when in doubt
6. Single protocol per cycle: one action only
7. Amount limits: never exceed idle balance
8. Deadline: max 10 minutes in the future
9. Bifrost priority: prefer SLP (safest), then DEX, then Farming
10. Impermanent loss: explicitly evaluate IL for DEX/farming positions

### Decision Pipeline

The prompt enforces a strict 3-step pipeline:

1. **INGEST** — receive market snapshot (yields, vault state, cross-chain state)
2. **ANALYZE** — compute net APY benefit after costs, assess risk per product category
3. **DECIDE** — output exactly one JSON decision object

## Running the Agent

```bash
# Install dependencies
pnpm install --filter @obidot/agent

# Type check
pnpm --filter @obidot/agent run typecheck

# Lint
pnpm --filter @obidot/agent run lint

# Start (dev mode with tsx watch)
pnpm --filter @obidot/agent run dev

# Start (production)
pnpm --filter @obidot/agent run start
```

The agent starts its autonomous loop and API server together. The loop runs one cycle at a configurable interval (default: 60 seconds). The API server is available at `http://localhost:3011` once started.
