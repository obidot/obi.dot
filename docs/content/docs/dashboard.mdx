The Obidot dashboard is a **Next.js 15** web application that gives operators a real-time view of vault performance, active strategies, cross-chain activity, and AI-powered analytics. It connects to the [AI Agent](/docs/agent) REST API and reads on-chain state via Wagmi.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Framework | Next.js 15 (App Router, Turbopack) |
| UI | React 19 |
| Styles | TailwindCSS v4 (custom utility classes) |
| Server state | TanStack Query v5 (polling every 10 s) |
| Wallet | Wagmi v2 + RainbowKit v2 + viem v2 |
| Charts | recharts (PnL candlestick chart) |
| Icons | lucide-react |

## Running Locally

```bash
# From the repo root
pnpm --filter @obidot/app run dev
```

The app runs on **port 3000** by default. It proxies all `/api/*` requests to the agent API on `http://localhost:3001/api/*` via `next.config.ts` rewrites, so the agent must be running for live data.

```bash
# Start the agent first (port 3001)
pnpm --filter @obidot/agent run dev

# Then start the dashboard (port 3000)
pnpm --filter @obidot/app run dev
```

## API Proxy

All data fetching goes through an API proxy route (`src/app/api/[...path]/route.ts`) that rewrites requests to the agent backend:

```
GET /api/vault        ŌåÆ http://localhost:3001/api/vault
GET /api/yields       ŌåÆ http://localhost:3001/api/yields
GET /api/strategies   ŌåÆ http://localhost:3001/api/strategies
GET /api/crosschain   ŌåÆ http://localhost:3001/api/crosschain
GET /api/agent        ŌåÆ http://localhost:3001/api/agent
POST /api/agent/trigger ŌåÆ http://localhost:3001/api/agent/trigger
```

This means the dashboard can be deployed behind a reverse proxy without exposing the agent port directly.

## Pages

### `/` ŌĆö Vault Dashboard

The landing page provides a high-level health check of the vault.

| Component | Description |
|-----------|-------------|
| `PageHero` | Navbar with live ticker strip (DOT/USD price), wallet connect pill |
| `VaultOverview` | Total assets, share price, remote assets, deposit cap utilization |
| `QuickStats` | Four KPI cards ŌĆö TVL, APY, active strategies, idle ratio |
| `PnlChart` | Candlestick chart (demo data ŌĆö not live P&L) |
| `VaultActions` | Deposit / Withdraw forms (UI stub ŌĆö no on-chain transaction yet) |
| `HealthIndicators` | Oracle freshness, circuit breaker status, emergency mode flag |

**Live data:** vault balances and share price are read directly from the contract via Wagmi (`useBalance`, `useReadContract`). The wallet connect pill shows the connected address's tDOT balance in real time.

### `/yields` ŌĆö Yield Opportunities

| Component | Description |
|-----------|-------------|
| `YieldGrid` | Responsive card grid of all known DeFi protocols |
| `ProtocolCard` | Shows protocol name, chain, current APY, TVL, risk tier |

APY data is fetched from the agent's `/api/yields` endpoint. The agent fetches TVL from DeFiLlama and simulates APY with a sine-wave oscillator (no live APY feed exists on Paseo testnet).

### `/strategies` ŌĆö Strategy History

| Component | Description |
|-----------|-------------|
| `StrategyTable` | Full strategy history ŌĆö sortable and filterable by status/chain/protocol |
| `StrategyDetail` | Slide-over side panel (with focus trap) showing full intent data and outcome |

Strategies are fetched from `/api/strategies` and polled every 10 seconds. Clicking a row opens the detail panel without a navigation change.

### `/crosschain` ŌĆö Cross-Chain Activity

| Component | Description |
|-----------|-------------|
| `ChainTopology` | Visual node graph showing Polkadot Hub connections to satellite chains |
| `SatelliteTable` | Per-chain table: TVL, last sync age, status |

Data is fetched from `/api/crosschain`.

### `/agent` ŌĆö Agent Status

| Component | Description |
|-----------|-------------|
| `AgentStatus` | Derived state: running / idle / error, last decision timestamp, loop iteration count |
| `DecisionFeed` | Scrollable list of recent agent decisions |
| `DecisionCard` | Per-decision: type (deploy / rebalance / hold / error), rationale text, timestamp |

The `AgentStatus` component derives its display state from `lastDecision` and `isRunning` fields ŌĆö it does not rely on a separate status endpoint.

### `/insights` ŌĆö Financial Intelligence Hub

The insights page is the most feature-rich section of the dashboard, providing AI-powered analytics across six components backed by three computation engines.

#### Components

| Component | Description |
|-----------|-------------|
| `MarketPulse` | Real-time market sentiment: DOT price, 24h change, volume trend, fear/greed indicator |
| `OpportunityRadar` | Ranked list of yield opportunities scored by the opportunity scoring engine |
| `RiskMatrix` | 2D risk/return scatter plot for all active positions and protocols |
| `PositionSimulatorPanel` | Interactive simulator: enter a position size and duration to project outcomes |
| `PortfolioOptimizer` | Suggests optimal allocation weights across protocols given risk tolerance |
| `YieldComparison` | Side-by-side APY comparison across protocols and chains |

#### Computation Engines

The three engines in `src/lib/` perform client-side calculations on top of data from the agent API:

**`yield-opportunity-score.ts`** ŌĆö Scores each protocol opportunity on a 0ŌĆō100 scale:
- Weighted factors: APY (40%), TVL stability (25%), risk tier (20%), liquidity depth (15%)
- Outputs a ranked list used by `OpportunityRadar`

**`risk-analyzer.ts`** ŌĆö Computes risk metrics for each position and protocol:
- Volatility score, liquidity risk, smart contract risk, cross-chain execution risk
- Outputs `(risk, return)` coordinates used by `RiskMatrix`

**`position-simulator.ts`** ŌĆö Projects position outcomes over time:
- Inputs: principal amount, duration (days), protocol selection
- Models: compound APY, gas costs, impermanent loss estimate, slippage
- Outputs: projected value, P&L, best/worst case scenarios

## Hooks

| Hook | Returns |
|------|---------|
| `useVaultData()` | Vault state (TVL, share price, cap, remote assets) |
| `useYields()` | Protocol yield opportunities array |
| `useStrategies()` | Strategy records array |
| `useAgentStatus()` | Agent running state and last decision |
| `useWebSocket()` | WebSocket connection (defined, currently unused in UI) |

All data-fetching hooks use TanStack Query with a 10-second `refetchInterval`.

## Known Gaps

| Gap | Detail |
|-----|--------|
| `PnlChart` uses demo data | Hard-coded candlestick data, not live P&L from the vault |
| `VaultActions` is a UI stub | Deposit/withdraw forms exist but do not submit on-chain transactions |
| `useWebSocket` unused | Hook defined in `src/hooks/useWebSocket.ts` but not wired to any component |
| `src/stores/` is empty | Zustand store directory reserved for future state management |
| APYs are simulated | The agent's yield service uses a sine-wave oscillator; no live APY feed on Paseo |
