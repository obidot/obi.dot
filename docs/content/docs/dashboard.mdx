The Obidot dashboard is a **Next.js 15** web application that gives operators a real-time view of vault performance, active strategies, cross-chain activity, and AI-powered analytics. It reads on-chain state live from Polkadot Hub TestNet via Wagmi, queries historical events from the [obi.index](/docs/architecture) GraphQL indexer, and proxies yield/crosschain/agent-log calls to the [AI Agent](/docs/agent) REST API.

## Tech Stack

| Layer        | Technology                                                            |
| ------------ | --------------------------------------------------------------------- |
| Framework    | Next.js 15 (App Router, Turbopack)                                    |
| UI           | React 19                                                              |
| Styles       | TailwindCSS v4 (custom utility classes)                               |
| Server state | TanStack Query v5 (polling every 10–30 s)                             |
| Wallet       | Wagmi v2 + RainbowKit v2 + viem v2                                    |
| GraphQL      | Raw `graphql-ws` for subscriptions; `fetchGraphQL()` for HTTP queries |
| Charts       | recharts (PnL candlestick chart)                                      |
| Icons        | lucide-react                                                          |

## Trade UI

The `/swap` (and `/:trade/:chain/:router`) routes render a two-panel trading interface.

### Swap Tab

- **Left panel:** Token picker (inline dropdown for both sides), amount input, quote display, execute button. Uses `SwapForm`.
- **Right panel:** `InfoBanners` (trending/farming pools ticker) + `RouteDiagram`. The route diagram shows on-chain routes (single/multi-hop, split), an adapter quotes comparison table, and cross-chain route stubs. Routes are sourced from `/api/routes` with a 600 ms debounce.

### Limit Order Tab

- **Left panel:** Token pair (via `TokenPicker`), sell amount, target price with ±% quick-set buttons and delta indicator, expiry selector. Submits a `PendingOrder` to `localStorage("obidot_limit_orders")` and dispatches a `CustomEvent("obidot:order-placed")`.
- **Right panel:** `OrdersPanel` — live list of active and expired orders. Cancel removes the order from localStorage. Orders are monitored by the Obidot AI Agent and executed via `UniversalIntent` when the target price is reached.

### Cross-Chain Tab

- **Left panel:** Source and destination token (both via `TokenPicker`), amount input, destination chain selector (XCM or Bridge). For **RelayTeleport**, the output is estimated at 1:1 minus ~0.1% XCM fee with no exchange rate. For other chains, amounts are shown when routes are available.
- **Right panel:** Idle (cross-chain positions panel is planned for a future iteration).

## Running Locally

```bash
# From the repo root
pnpm --filter @obidot/app run dev
```

The app runs on **port 3010** by default. It proxies `/api/*` to the agent API on `http://localhost:3001/api/*` via `next.config.ts` rewrites; the agent must be running for yields/crosschain/agent-log data.

```bash
# Start the agent first (port 3001, default API_PORT)
pnpm --filter @obidot/agent run dev

# Then start the dashboard (port 3010)
pnpm --filter @obidot/app run dev
```

## Data Sources

The dashboard uses three distinct data sources:

| Source                                  | Used For                                                                                                                       |
| --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| **On-chain (Wagmi `useReadContracts`)** | Live vault state: `totalAssets`, `totalSupply`, `paused`, `emergencyMode`, `totalRemoteAssets`, user share balance             |
| **obi.index GraphQL**                   | Historical events: deposits, withdrawals, swap executions, strategy executions, vault stats — queried via `src/lib/graphql.ts` |
| **Agent REST API (proxied `/api/*`)**   | Yield opportunities, cross-chain topology, agent decisions, vault performance                                                  |

No Apollo Client is used — GraphQL subscriptions use raw `graphql-ws`; HTTP queries use the `fetchGraphQL<T>()` helper.

## API Proxy

`next.config.ts` rewrites proxy yield/crosschain/agent routes to the agent backend:

```
GET /api/yields            → http://localhost:3001/api/yields
GET /api/crosschain/state  → http://localhost:3001/api/crosschain/state
GET /api/agent/log         → http://localhost:3001/api/agent/log
POST /api/chat             → http://localhost:3001/api/chat
GET /api/strategies        → http://localhost:3001/api/strategies
GET /api/routes            → http://localhost:3001/api/routes
GET /api/swap/quote        → http://localhost:3001/api/swap/quote
GET /api/vault/state       → http://localhost:3001/api/vault/state
```

## Pages

### `/[trade]/[chain]/[router]` — DEX Aggregator (Trade)

The primary trading interface. URL segments encode the trade mode, chain, and token pair:

```
/swap/polkadot-hub-testnet/tdot-to-tusdc
/limit/polkadot-hub-testnet/tdot-to-tusdc
/crosschain/polkadot-hub-testnet/tdot-to-tusdc
```

| Component             | Description                                                                                    |
| --------------------- | ---------------------------------------------------------------------------------------------- |
| `SwapPanel`           | Left column: tab bar (Swap / Limit / Cross-Chain) + active form                                |
| `SwapForm`            | Token inputs, live quote via on-chain `SwapQuoter`, approve-then-swap state machine            |
| `LimitOrderPanel`     | Target price input, market delta indicator, expiry selector, localStorage order persistence    |
| `CrossChainSwapPanel` | XCM/bridge destination selector, live route from `/api/routes`, status-aware execute button    |
| `RouteDiagram`        | Right column: per-adapter quote comparison table, split-route toggle (60/40 weight sliders)    |

**Wallet redirect:** when the connected wallet switches chains, `TradePage` auto-redirects the URL chain segment to match (e.g. `polkadot-hub-testnet` → `polkadot-hub`).

**Split execution:** selecting two routes in the diagram enables `swapSplit()` with configurable BPS weights summing to 10000.

### `/` — Vault Dashboard

The landing page provides a high-level health check of the vault.

| Component          | Description                                                                               |
| ------------------ | ----------------------------------------------------------------------------------------- |
| `PageHero`         | Navbar with live ticker strip (DOT/USD price), wallet connect pill                        |
| `VaultOverview`    | Total assets, share price, remote assets, deposit cap utilization                         |
| `QuickStats`       | Four KPI cards — TVL, APY, active strategies, idle ratio                                  |
| `PnlChart`         | Candlestick chart (demo data — not live P&L)                                              |
| `VaultActions`     | Deposit / Withdraw forms with real on-chain transactions (approve → deposit step machine) |
| `HealthIndicators` | Circuit breaker (`paused`) and emergency mode flag, read live from chain                  |
| `RecentActivity`   | Inline feed of last 5 deposits + 5 swaps from obi.index                                   |

**Live data:** vault balances and share price are read directly from the contract via Wagmi `useReadContracts`. Deposit and withdraw execute real on-chain transactions using `useWriteContract` with an approve-then-deposit state machine.

### `/yields` — Yield Opportunities

| Component      | Description                                             |
| -------------- | ------------------------------------------------------- |
| `YieldGrid`    | Responsive card grid of all known DeFi protocols        |
| `ProtocolCard` | Shows protocol name, chain, current APY, TVL, risk tier |

APY data is fetched from the agent's `/api/yields` endpoint. The agent fetches TVL from DeFiLlama and simulates APY with a sine-wave oscillator (no live APY feed exists on Paseo testnet).

### `/strategies` — Strategy History

| Component        | Description                                                                  |
| ---------------- | ---------------------------------------------------------------------------- |
| `StrategyTable`  | Full strategy history — sortable and filterable by status/chain/protocol     |
| `StrategyDetail` | Slide-over side panel (with focus trap) showing full intent data and outcome |

Strategies are fetched from obi.index (`getIndexedStrategyExecutions`) and polled every 30 seconds. Clicking a row opens the detail panel without a navigation change.

### `/crosschain` — Cross-Chain Activity

| Component        | Description                                                            |
| ---------------- | ---------------------------------------------------------------------- |
| `ChainTopology`  | Visual node graph showing Polkadot Hub connections to satellite chains |
| `SatelliteTable` | Per-chain table: TVL, last sync age, status                            |

Data is fetched from `/api/crosschain`.

### `/agent` — Agent Status

| Component      | Description                                                                          |
| -------------- | ------------------------------------------------------------------------------------ |
| `AgentStatus`  | Derived state: running / idle / error, last decision timestamp, loop iteration count |
| `DecisionFeed` | Scrollable list of recent agent decisions                                            |
| `DecisionCard` | Per-decision: type (deploy / rebalance / hold / error), rationale text, timestamp    |

The `AgentStatus` component derives its display state from `lastDecision` and `isRunning` fields — it does not rely on a separate status endpoint.

### `/insights` — Financial Intelligence Hub

The insights page is the most feature-rich section of the dashboard, providing AI-powered analytics across six components backed by three computation engines.

#### Components

| Component                | Description                                                                           |
| ------------------------ | ------------------------------------------------------------------------------------- |
| `MarketPulse`            | Real-time market sentiment: DOT price, 24h change, volume trend, fear/greed indicator |
| `OpportunityRadar`       | Ranked list of yield opportunities scored by the opportunity scoring engine           |
| `RiskMatrix`             | 2D risk/return scatter plot for all active positions and protocols                    |
| `PositionSimulatorPanel` | Interactive simulator: enter a position size and duration to project outcomes         |
| `PortfolioOptimizer`     | Suggests optimal allocation weights across protocols given risk tolerance             |
| `YieldComparison`        | Side-by-side APY comparison across protocols and chains                               |

#### Computation Engines

The three engines in `src/lib/` perform client-side calculations on top of data from the agent API:

**`yield-opportunity-score.ts`** — Scores each protocol opportunity on a 0–100 scale:

- Weighted factors: APY (40%), TVL stability (25%), risk tier (20%), liquidity depth (15%)
- Outputs a ranked list used by `OpportunityRadar`

**`risk-analyzer.ts`** — Computes risk metrics for each position and protocol:

- Volatility score, liquidity risk, smart contract risk, cross-chain execution risk
- Outputs `(risk, return)` coordinates used by `RiskMatrix`

**`position-simulator.ts`** — Projects position outcomes over time:

- Inputs: principal amount, duration (days), protocol selection
- Models: compound APY, gas costs, impermanent loss estimate, slippage
- Outputs: projected value, P&L, best/worst case scenarios

## Hooks

| Hook                       | Returns                                                                                                                                        |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| `useVaultState()`          | Live on-chain `VaultOnChainState` — `totalAssets`, `totalSupply`, `totalRemoteAssets`, `paused`, `emergencyMode`, `tvlDisplay` (bigint fields) |
| `useShareBalance(address)` | User's share balance and max withdrawable assets                                                                                               |
| `useIndexedVaultState()`   | Historical vault state snapshot from obi.index                                                                                                 |
| `useVaultStats()`          | Aggregated stats from obi.index (strategy count, deposit/swap totals)                                                                          |
| `useVaultPerformance()`    | Cumulative PnL and high-water mark from agent API (legacy)                                                                                     |
| `useYields()`              | Protocol yield opportunities from agent API                                                                                                    |
| `useBifrostYields()`       | Bifrost-specific yield opportunities from agent API                                                                                            |
| `useStrategies()`          | Strategy execution records from obi.index (`IndexedStrategyExecution[]`)                                                                       |
| `useAgentLog()`            | Agent decision log from agent API                                                                                                              |

On-chain hooks use Wagmi `useReadContracts` with a 12-second `refetchInterval` (≈ one block). GraphQL hooks use TanStack Query with a 30-second `staleTime`.

## Known Gaps

| Gap                       | Detail                                                                                               |
| ------------------------- | ---------------------------------------------------------------------------------------------------- |
| `PnlChart` uses demo data | Hard-coded candlestick data, not live P&L from the vault                                             |
| APYs are simulated        | The agent's yield service uses a sine-wave oscillator; no live APY feed on Paseo testnet             |
| Limit orders are off-chain | `/limit` tab stores orders in `localStorage` — the agent monitors and executes them via UniversalIntent when price is reached, but there is no on-chain limit order contract |
| Price oracle feed         | DOT/USD market price in `MarketPulse` uses DeFiLlama/CoinGecko; `KeeperOracle` feeds are on-chain but not yet surfaced in the UI |
