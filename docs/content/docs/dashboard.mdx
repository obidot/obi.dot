The Obidot dashboard is a **Next.js 15** web application that gives operators and users a real-time view of swaps, limit orders, cross-chain activity, agent proposals, and analytics. It reads on-chain state live from Polkadot Hub TestNet via Wagmi, queries historical events from the [obi.index](/docs/architecture) GraphQL indexer, and proxies chat, routing, and operational calls to the [AI Agent](/docs/agent) REST API.

## Trade UI

The `/swap` (and `/:trade/:chain/:router`) routes render a two-panel trading interface.

### Swap Tab

- **Left panel:** Token picker (inline dropdown for both sides), amount input, quote display, execute button. Uses `SwapForm`.
- **Right panel:** `InfoBanners` (trending/farming pools ticker) + `RouteDiagram`. The route diagram shows on-chain routes (single/multi-hop, split), an adapter quotes comparison table, and cross-chain route stubs. Routes are sourced from `/api/routes` with a 600 ms debounce.

### Limit Order Tab

- **Left panel:** Token pair (via `TokenPicker`), sell amount, target price with Â±% quick-set buttons and delta indicator, expiry selector. Submits to the agent's `/api/limit-orders` surface instead of localStorage.
- **Right panel:** `OrdersPanel` â€” server-backed order list with Pending, Triggered, Filled, Expired, and Cancelled states. Triggered orders can auto-open a review modal and reuse the same approve-then-swap wallet flow as chat proposals.

### Cross-Chain Tab

- **Left panel:** Source and destination token (both via `TokenPicker`), amount input, destination chain selector (XCM or Bridge), and route-status badges that distinguish `live`, `simulated`, `mainnet_only`, and `coming_soon`.
- **Right panel:** Pair-aware reachability plus `CrossChainStatusPanel`, which shows indexed lifecycle steps for recent pipelines associated with the connected wallet when sender attribution is available. The current tracker is strongest for locally indexed Polkadot Hub lifecycle rows; remote destination-host receipts are still a known gap.

## Running Locally

```bash
# From the repo root
pnpm --filter @obidot/app run dev
```

The app runs on **port 3010** by default. It proxies `/api/*` to the agent API on `http://localhost:3011/api/*` via `next.config.ts` rewrites; the agent must be running for yields, cross-chain state, swap routing, and decision-log data.

```bash
# Start the agent first (port 3011, default API_PORT)
pnpm --filter @obidot/agent run dev

# Then start the dashboard (port 3010)
pnpm --filter @obidot/app run dev
```

## Data Sources

The dashboard uses three distinct data sources:

| Source                                  | Used For                                                                                                                       |
| --------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| **On-chain (Wagmi `useReadContracts`)** | Live vault state: `totalAssets`, `totalSupply`, `paused`, `emergencyMode`, `totalRemoteAssets`, user share balance             |
| **obi.index GraphQL**                   | Historical events, protocol analytics, cross-chain pipelines, and price history â€” queried via `src/lib/graphql.ts` |
| **Agent REST API (proxied `/api/*`)**   | Chat streaming, limit orders, route discovery, yields, cross-chain topology, agent decisions, vault performance    |

No Apollo Client is used â€” GraphQL subscriptions use raw `graphql-ws`; HTTP queries use the `fetchGraphQL<T>()` helper.

### `/insights` â€” Financial Intelligence Hub

The insights page is the most feature-rich section of the dashboard, providing AI-powered analytics across six components backed by three computation engines.

#### Components

| Component                | Description                                                                           |
| ------------------------ | ------------------------------------------------------------------------------------- |
| `ProtocolAnalyticsPanel` | Indexed stat cards plus top-routes table backed by `protocolStats` and `topRoutes`    |
| `PriceHistoryPanel`      | Hourly OHLCV chart with pair/window selectors backed by `priceHistory`                 |
| `MarketPulse`            | Real-time market sentiment: DOT price, 24h change, volume trend, fear/greed indicator |
| `OpportunityRadar`       | Ranked list of yield opportunities scored by the opportunity scoring engine           |
| `RiskMatrix`             | 2D risk/return scatter plot for all active positions and protocols                    |
| `PositionSimulatorPanel` | Interactive simulator: enter a position size and duration to project outcomes         |
| `PortfolioOptimizer`     | Suggests optimal allocation weights across protocols given risk tolerance             |
| `YieldComparison`        | Side-by-side APY comparison across protocols and chains                               |

#### Computation Engines

The three engines in `src/lib/` perform client-side calculations on top of data from the agent API:

**`yield-opportunity-score.ts`** â€” Scores each protocol opportunity on a 0â€“100 scale:

- Weighted factors: APY (40%), TVL stability (25%), risk tier (20%), liquidity depth (15%)
- Outputs a ranked list used by `OpportunityRadar`

**`risk-analyzer.ts`** â€” Computes risk metrics for each position and protocol:

- Volatility score, liquidity risk, smart contract risk, cross-chain execution risk
- Outputs `(risk, return)` coordinates used by `RiskMatrix`

**`position-simulator.ts`** â€” Projects position outcomes over time:

- Inputs: principal amount, duration (days), protocol selection
- Models: compound APY, gas costs, impermanent loss estimate, slippage
- Outputs: projected value, P&L, best/worst case scenarios

## Known Gaps

| Gap                       | Detail                                                                                               |
| ------------------------- | ---------------------------------------------------------------------------------------------------- |
| PnL chart still uses demo data | The dedicated price history panel is live, but vault PnL candlesticks are still not fully indexed |
| APYs are simulated        | The agent's yield service still uses simulated APY output on the current Polkadot Hub TestNet setup   |
| Limit orders are still off-chain | The agent monitors and persists them server-side, but there is still no on-chain limit order contract |
| Price oracle feed         | DOT/USD market price in `MarketPulse` uses DeFiLlama/CoinGecko; `KeeperOracle` feeds are on-chain but not yet surfaced in the UI |
| Remote chain receipts     | Cross-chain lifecycle tracking is based on locally indexed router/executor events; destination-host execution on remote chains is not yet indexed as a separate source |
