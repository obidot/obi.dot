Welcome to the Obidot documentation. Obidot is the **first DEX aggregator on Polkadot Hub** — a non-custodial protocol that aggregates liquidity across the Polkadot ecosystem and connected EVM chains. Users deposit DOT/USDC into an ERC-4626 vault; trades are routed cross-chain via XCM (to Polkadot parachains) and Hyperbridge ISMP (to Ethereum, Arbitrum, Base); an autonomous AI agent operates as a **sub-feature** for intent-based strategy execution.

## Overview

Obidot is built around a single entry point on Polkadot Hub:

```
User → ObidotVault (ERC-4626)
             │
    ┌────────┼───────────────┐
    ▼        ▼               ▼
SwapRouter  XCMExecutor  HyperExecutor
(on-hub)   (parachains)  (EVM chains)
    │           │              │
 Hydration   Bifrost(2030)  Ethereum
 AssetHub    Acala(2000)    Arbitrum
 BifrostDEX  Moonbeam(2004) Base
```

| Component                    | What it does                                                                                                            |
| ---------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| **SwapRouter**               | Aggregates on-hub liquidity — single-hop, multi-hop, split-route across Hydration Omnipool, AssetHub pairs, Bifrost DEX |
| **XCMExecutor**              | Dispatches trades to Polkadot parachains via native XCM V4                                                              |
| **HyperExecutor**            | Bridges to EVM chains (Ethereum, Arbitrum, Base) via Hyperbridge ISMP                                                   |
| **ObidotVault**              | ERC-4626 vault: deposit DOT/USDC, earn yield from routed strategies, enforces policy engine                             |
| **AI Agent** _(sub-feature)_ | Signs EIP-712 strategy intents off-chain for automated execution — relayed permissionlessly on-chain                    |

## Key Features

- **First DEX Aggregator on Polkadot Hub** — Pluggable pool adapter architecture routing swaps across Hydration Omnipool, AssetHub native pairs, Bifrost DEX, and future on-hub AMMs
- **Cross-Chain Liquidity** — Native XCM V4 to Polkadot parachains (Bifrost, Acala, HydraDX, Moonbeam) and Hyperbridge ISMP to EVM chains
- **ERC-4626 Compliant Vault** — Standard yield-bearing vault with deposit caps, inflation-attack mitigation via virtual share offset, and conservative rounding
- **Satellite Vaults on EVM** — `ObidotVaultEVM` on Ethereum/Arbitrum/Base syncs global share price via Hyperbridge ISMP
- **Bifrost DeFi Strategies** — Liquid staking (SLP/vDOT), DEX swaps, yield farming, and SALP via on-chain Bifrost adapter with SCALE-encoded XCM calls
- **Oracle-Based Slippage Protection** — `KeeperOracle` (Chainlink-compatible) with multi-asset registry and 2% max-slippage guard
- **On-Chain Policy Engine** — Parachain and protocol whitelists, per-protocol exposure caps, daily loss circuit breaker
- **AI Agent Sub-Feature** — LangChain/GPT-4o agent signs EIP-712 intents for multi-hop strategies; anyone can relay valid intents (permissionless)
- **Real-Time Dashboard** — Next.js 15 app with vault overview, yield grid, strategy feed, cross-chain topology, agent status, and Financial Intelligence Hub
- **Emergency Mode** — Admin can pause the vault and enable proportional withdrawals that bypass remote asset accounting

## Tech Stack

### Smart Contracts

- **Solidity** `0.8.28` with Foundry (631 tests, 97 suites)
- **OpenZeppelin Contracts** v5.5.0 (`ERC4626`, `AccessControl`, `Pausable`, `ReentrancyGuard`)
- **Pyth SDK Solidity** v2.2.0 for oracle interface
- **PolkaVM** deployment via Revive LLVM compiler (`resolc`)

### Off-Chain Agent

- **Node.js / TypeScript** — strict ES2022, pino logging
- **LangChain** (`@langchain/core`, `@langchain/openai`) — GPT-4o agent loop
- **viem** v2 — on-chain reads and EIP-712 signing
- **Fastify** — REST API server on port 3001
- **Telegram Bot API** — optional operator notifications

### Dashboard App

- **Next.js 15** (App Router, Turbopack) with **React 19**
- **TailwindCSS v4** — custom design system
- **TanStack Query v5** — server state and polling
- **Wagmi v2** + **RainbowKit v2** + **viem v2** — wallet connection and on-chain reads
