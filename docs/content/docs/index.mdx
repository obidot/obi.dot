Welcome to the Obidot documentation. Obidot is an **Autonomous Cross-Chain Finance Layer** for Polkadot, built as an ERC-4626 yield-bearing vault deployed to the Polkadot Hub EVM (REVM). It enables an off-chain AI agent to autonomously route funds across Polkadot parachains via native XCM, while enforcing strict on-chain risk policies.

## Overview

Obidot separates concerns across three distinct actor types:

| Role | Responsibility |
|------|---------------|
| **Admin** | Configures policy engine parameters, manages roles, emergency controls |
| **AI Strategist** | Signs EIP-712 strategy intents specifying capital deployment targets |
| **Keeper** | Reports cross-chain strategy outcomes and P&L back to the vault |

Depositors interact with a standard ERC-4626 interface — `deposit`, `mint`, `withdraw`, and `redeem` — while the AI strategist operates within on-chain policy bounds enforced by the smart contract itself.

## Key Features

- **ERC-4626 Compliant Vault** — Standard yield-bearing vault with deposit caps, inflation-attack mitigation via virtual share offset, and conservative rounding
- **EIP-712 Signed Strategy Intents** — The AI agent signs typed data off-chain; anyone can relay valid intents on-chain (permissionless relaying)
- **On-Chain Policy Engine** — Parachain and protocol whitelists, per-protocol exposure caps, daily loss circuit breaker
- **Oracle-Based Slippage Protection** — Pyth Network price feeds validate expected returns against real-time prices
- **Native XCM Dispatch** — Cross-chain messages sent via the Polkadot Hub XCM precompile at `0x00000000000000000000000000000000000a0000`
- **DEX Aggregator Router** — On-hub swap routing via `SwapRouter` with pluggable pool adapters (Hydration Omnipool, AssetHub pairs, Bifrost DEX)
- **Bifrost DeFi Strategies** — Liquid staking (SLP), DEX swaps, yield farming, and SALP via on-chain Bifrost adapter with SCALE-encoded XCM calls
- **Autonomous AI Agent** — LangChain-powered agent with on-chain perception tools, Bifrost yield analysis, cross-chain rebalancing, Fastify REST API, and Telegram bot
- **Financial Intelligence Hub** — Next.js dashboard with AI-powered market pulse, opportunity radar, risk matrix, position simulator, and portfolio optimizer
- **Real-Time Dashboard** — Six-page web app (Next.js 15, React 19, TailwindCSS v4) with live wallet balance, strategy feed, and cross-chain topology
- **Emergency Mode** — Admin can pause the vault and enable proportional withdrawals that bypass remote asset accounting

## Tech Stack

### Smart Contracts
- **Solidity** `0.8.28` with Foundry (562 tests, 97 suites)
- **OpenZeppelin Contracts** v5.5.0 (`ERC4626`, `AccessControl`, `Pausable`, `ReentrancyGuard`)
- **Pyth SDK Solidity** v2.2.0 for oracle interface
- **PolkaVM** deployment via Revive LLVM compiler (`resolc`)

### Off-Chain Agent
- **Node.js / TypeScript** — strict ES2022, pino logging
- **LangChain** (`@langchain/core`, `@langchain/openai`) — GPT-4o agent loop
- **viem** v2 — on-chain reads and EIP-712 signing
- **Fastify** — REST API server on port 3001
- **Telegram Bot API** — optional operator notifications

### Dashboard App
- **Next.js 15** (App Router, Turbopack) with **React 19**
- **TailwindCSS v4** — custom design system
- **TanStack Query v5** — server state and polling
- **Wagmi v2** + **RainbowKit v2** + **viem v2** — wallet connection and on-chain reads
